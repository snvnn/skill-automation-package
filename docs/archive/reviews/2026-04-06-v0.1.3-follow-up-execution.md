## Completed Changes

### 1. Python 3.10 compatibility at install time and runtime

Rationale:

- the observed install session failed under a default `python3` that resolved to Python 3.10
- the installed runtime asset needed the same compatibility treatment as the installer itself

Implemented:

- added `timezone.utc` fallback handling to `scripts/install.py`
- added the same fallback to `assets/.claude/tools/skill_agent.py`
- added the same fallback to `assets/.claude/tests/test_skill_agent.py`
- updated the README prerequisite from Python 3.11+ to Python 3.10+

### 2. Truthful dry-run output

Rationale:

- preview output must not look like a completed write when no file mutation happened

Implemented:

- changed installer dry-run labels to `Would update ...` and `Would write ...`
- added CLI assertions for dry-run output in `tests/test_install.py`

### 3. Target-repo git hygiene guidance

Rationale:

- downstream users need an explicit policy for what to commit and what to ignore after installation

Implemented:

- documented shared-versus-local installation policies in the README
- included example `.gitignore` patterns for generated metadata and local-only installs

### 4. Reinstall and upgrade integration coverage

Rationale:

- the package needed proof that reinstalling over an existing target repo behaves the way the docs claim

Implemented:

- added a CLI-level reinstall regression test in `tests/test_install.py`
- covered:
  - overwriting shipped files
  - replacing managed blocks without duplicating them
  - preserving `usage.json`
  - preserving user-created local skills
  - regenerating `registry.json`
  - leaving stale orphaned files in place when the package no longer ships them

### 5. Clear default-versus-skip-docs upgrade paths

Rationale:

- operators were able to skim the old upgrade section and mistake the skip-docs form for the default path

Implemented:

- split the README into `Default Upgrade Path` and `Upgrade Without Managed Docs`
- added guidance-content tests so both sections remain distinct

### 6. `sync_assets.py` edge-case coverage

Rationale:

- the package had tests for explicit roots and basic ancestor auto-detection, but not for competing matches

Implemented:

- added tests for direct-parent detection
- added tests for multiple matching ancestors
- documented and locked the current rule: choose the nearest matching parent

### 7. Verification commands now match the bytecode-clean recommendation

Rationale:

- the README still showed raw unittest commands even though the package already favored `PYTHONDONTWRITEBYTECODE=1`

Implemented:

- updated README verification examples to use `PYTHONDONTWRITEBYTECODE=1`
- added README guidance tests for those commands
